## 測定しないものは改善できない

ピーター・ドラッカーの有名な言葉に「測定できないものは管理できない」がある。

個人開発の収益化においても同じだ。PVを測定していなければ、どのサイトが伸びているかわからない。検索順位を追っていなければ、SEO施策が効いているかわからない。**感覚で「なんとなくうまくいってない」と思っている間に、改善のタイミングを逃す。**

`/revenue-check` は、この測定と分析を自動化するスキルだ。

## チェック項目

実行すると、Claude Codeが以下の5つの項目を順番にチェックする。

### 1. サイト稼働確認

全サイトにHTTPリクエストを送り、200 OKが返るかを確認する。

```
https://devtools-jp-liard.vercel.app  → 200 OK ✅
https://qrcode-jp.vercel.app          → 200 OK ✅
https://passgen-jp.vercel.app          → 200 OK ✅
https://imgconv-jp.vercel.app          → 200 OK ✅
https://mojicount-jp.vercel.app        → 200 OK ✅
https://privacy-jp.vercel.app          → 200 OK ✅
```

Vercel無料枠はほぼダウンしないが、設定ミスやデプロイエラーで落ちることはある。定期チェックで早期発見する。

### 2. 収益確認

Buy Me a Coffeeの支援状況をオーナーに確認する。将来的にAdSenseが入れば、そのダッシュボード情報も確認対象になる。

**ここがスキルの限界でもある。** Claude Codeはブラウザでログインしてダッシュボードを見ることはできない。オーナーに「Buy Me a Coffeeの支援はありましたか？」と聞くことしかできない。

この限界は認識した上で、「定期的に確認する仕組みがあること自体」に価値がある。

### 3. アクセス状況確認

以下のデータ元をオーナーに確認する：

- **Vercel Analytics** — 各サイトのPV数
- **Google Search Console** — 表示回数、クリック数、掲載順位
- **どのサイト・キーワードが伸びているか**

### 4. SEO状況

ここはClaude Codeが自律的に実行できる。Web検索でインデックス状況を確認する。

```
検索: site:qrcode-jp.vercel.app
結果: 0件 → 未インデックス ❌
```

ターゲットキーワードでの検索順位も確認する。

```
検索: "QRコード 作成 無料"
結果: qrcode-jp.vercel.app は上位100件に含まれず ❌
```

これにより、SEO施策の効果を客観的に測定できる。

### 5. 競合分析

ターゲットキーワードで上位表示されている競合サイトを調査する。

例えば「QRコード 作成 無料」で検索すると、qr.quel.jp、Adobe Express、cman.jpなどの強力な競合が見つかる。

**競合が強すぎる場合は、そのキーワードを捨てる判断も重要。** ニッチなキーワード（例: 「QRコード 名刺用 無料」）にピボットするか、別のサイトに注力するかを判断する材料になる。

## レポートの形式

チェック結果は以下の形式でレポートに保存される。

```markdown
## 収益レポート（2026-03-09）

### 収益
| 収益源 | 今期 | 累計 |
|--------|------|------|
| Buy Me a Coffee | ¥0 | ¥0 |
| AdSense | 未導入 | — |

### サイト状況
| サイト | 稼働 | インデックス | 改善アクション |
|--------|------|------------|--------------|
| QRコード作成 | ✅ | ❌ 未 | SEO強化・コンテンツ拡充 |
| パスワード生成 | ✅ | ❌ 未 | キーワード強化 |

### 次のアクション（優先度順）
1. [最優先] Google Search Console登録
2. [基盤強化] コンテンツ拡充
3. [運用改善] Google Analytics導入
```

## 判断基準

レポートの数値に基づき、以下の判断基準で次のアクションを決定する。

| 状況 | 判断 |
|------|------|
| PVが伸びているサイトがある | → そのサイトのコンテンツ拡充・収益化強化を優先 |
| PVが低いサイトがある | → SEO改善か、損切りして別の手段にリソースを振る |
| 特定キーワードで順位上昇中 | → そのキーワード周辺の関連ツールを追加 |
| 全体的にPVが伸びない | → サイト量産戦略を見直し、別の収益手段を検討 |

最後の判断が最も重要だ。**全体的にうまくいっていない場合、「もっと頑張る」のではなく、「手段を変える」という判断ができる。**

## レポートの蓄積

レポートは `reports/YYYY-MM-DD-revenue.md` の形式で蓄積される。

```
reports/
├── 2026-03-09-revenue.md
├── 2026-03-10-ideas.md
└── 2026-03-10-article-plan.md
```

複数のレポートが蓄積されることで、**時系列での傾向分析**が可能になる。

- 先月と比べてPVは増えているか？
- Search Consoleのインデックス数は増えているか？
- どの施策が効いて、どの施策が効かなかったか？

この傾向分析こそが、戦略を軌道修正する根拠になる。

## /money-ideas との使い分け

| スキル | 目的 | 頻度 |
|--------|------|------|
| `/revenue-check` | 現状の把握と改善 | 週1回程度 |
| `/money-ideas` | 新しい手段の探索 | 月1回、または行き詰まった時 |

`/revenue-check` は「今やっていることは効いているか？」を確認するスキル。
`/money-ideas` は「他にもっと良い方法はないか？」を探索するスキル。

両方を定期的に実行することで、**現在の戦略の最適化**と**戦略の転換**の両方をカバーする。
